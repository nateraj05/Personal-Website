02 / FULL-STACK · VISA INC. · SUMMER 2026
TranView — 3DS Transaction Trace Debugger
Internal developer tooling for Open VisaNet (OVN)
COMPLETED
Built an internal full-stack developer tool for Visa engineers to debug 3D Secure transaction flows across Open VisaNet's distributed microservices architecture. TranView ingests raw logs from OpenSearch, runs them through a deterministic parsing pipeline, and uses a GenAI agent to classify each subsystem's behavior — rendering the full trace as an interactive "subway map" with live SSE streaming, a filterable log pane, and structured error panels.
What I Built (Backend)
  • Data pipeline — OpenSearch _msearch adapter with window-widening (24h → 7d), 50k-line cap, retry + circuit-breaker, and PII masking
  • Parser & validator — regex tokenizer, shape-hash dedup, closed-enum + monotonic-timestamp validation
  • GenAI agent — structured-output classification of subsystem shapes via Visa-internal LLM client; 3-attempt validate-and-repair loop; graceful degraded mode on outage
  • Multi-app profiles — profile.yaml + event_catalog.yaml loader; pipeline takes profile as input with no hard-coded subsystem strings
  • SSE streaming endpoint — GET /api/{appId}/trace/{traceKey}/stream; OTel spans + metrics tagged by app_id
What I Built (Frontend)
  • Subway map UI — interactive stage rail + subsystem chips with hover tooltips, status colors, and decision · score · latency captions
  • Detail panel — right-rail slide-in with stat grid, LLM summary/reasoning, focus trap + Esc close
  • Log pane — virtualized with react-virtuoso; filter by node/level/free-text; event chips link to emitting log lines
  • Live streaming — useTraceStream hook applies JSON Patches via SSE; opt-in via ?live=1
  • Error states + a11y — 6 typed error panels, theme toggle (light/dark/blueprint), ARIA labels, prefers-reduced-motion
